################################
# Amodal COCO-Detectron format #
################################

The .json-annotation-files in this archive have a format similar to the usual  
COCO-format (cocodataset.org). Namely, for each annotation, we have the        
following fields:                                                              
                                                                               
id:              Unique ID of this annotation                                  
image_id:        Image ID of the image where this annotation occurs            
category_id:     Category ID of the category of this annotation                
bbox:            Bounding box coordinates of this annotation                   
segmentation:    Polygon or RLE of the corresponding region                    
area:            Area of the 'segmentation'                                    
iscrowd:         Boolean indicating if this is a crowd-annotation              
                                                                               
However, in order to provide amodal annotations we changed/added several       
additional fields for visible and invisible masks, the occlusion level and     
occlusion rate:                                                                
                                                                               
id:              Unique ID of this annotation                                  
image_id:        Image ID of the image where this annotation occurs            
category_id:     Category ID of the category of this annotation                
bbox:            Bounding box coordinates of the amodal region of this         
                 annotation                                                    
segmentation:    RLE of the corresponding amodal region                        
area:            Area of the 'segmentation' (amodal region)                    
iscrowd:         Boolean indicating if this is a crowd-annotation              
                 (D2S has no crowd annoations)                                 
visible_mask:    RLE of the corresponding visible region                       
invisible_mask:  RLE of the corresponding visible region (optional, only
                 present if the object is occluded)                            
occlude_rate:    Rate of occlusion for this object                             
                 (= area(invisible_mask)/area(segmentation))                   
occl_depth:      Depth of the object, 0 is foreground, 1 if occluded by an     
                 object of occl_depth 0, 2 if occluded by an object of         
                 occl_depth 1, and so on...                                    
amodal_base:     Some optional additional information of the corresponding     
                 image                                                         
amodal_region:   Some optional additional information of the annotation

##########
# SPLITS #
##########

- COCO_amodal_*_detectron.json --> COCOA annotations as from Zhu et. al but
  converted to Amodal COCO-Detectron format
- COCO_amodal_*_no_stuff_detectron.json --> COCOA annotations as from Zhu et.
  al without 'stuff' annotations but converted to Amodal COCO-Detectron format
- COCO_amodal_*_with_classes.json --> COCOA cls annotations which have been
  generated by merging original COCO annotations with COCOA no stuff
  annotations as described in our work.
